{% trans "Projektor" %}

{% trans "Projektor aktualisiert sich automatisch" %}

{% blocktrans %}Änderung können vorgenommen werden ohne den Präsentationsmodus zu beenden. Eine verkleinerte Projektor-Live-Ansicht zeigt immer die aktuell ausgewählte Folie an.{% endblocktrans %}

{% trans "Dashboard mit Widgets" %}

{% blocktrans %}Das Dashboard ist die "Projektor-Steuereinheit" für die Versammlungsleitung. Alle Inhalte sind als "Folie" projizierbar und werden in flexiblen Widgets dargestellt.{% endblocktrans %}

{% trans "Countdown zur Redezeitbegrenzung" %}

{% blocktrans %}Zur Begrenzung der Redezeit lässt sich jederzeit ein konfigurierbarer Countdown einblenden. Zusätzlich ist stets die aktuelle Uhrzeit auf der Projektoransicht sichtbar.{% endblocktrans %}

{% trans "Eigene Folien erstellen" %}

{% blocktrans %}Erstellen Sie Folien für organisatorische Dinge abseits der formalen Tagesordnung.{% endblocktrans %}

{% trans "Rednerliste einblenden" %}

{% blocktrans %}Rednerlisten eines Tagesordnungspunktes lassen sich projizieren oder dezent in der unteren Ecke einblenden.{% endblocktrans %}

{% trans "Überblendungen" %}

{% blocktrans %}Kurze Meldungen lassen sich auf Folien überblenden. Die aktuelle Folie wird dabei nicht verlassen.{% endblocktrans %}




{% trans "Tagesordnung" %}

  • ↑ {% trans "Nach oben" %}
{% trans "Tagesordnung verwalten" %}

{% blocktrans %}Verwalten Sie Ihre Tagesordnung mit OpenSlides. Tagesordnungspunkte lassen sich (per CSV) importieren und leicht per Drag&Drop zu einer hierarchischen Liste mit Unterpunkten verschieben.{% endblocktrans %}

{% trans "Einträge projizieren" %}

{% blocktrans %}Jeder Tagesordnungspunkt kann per Mausklick auf dem Projektor angezeigt werden. Egal in welcher Reihenfolge! Auch Rednerlisten lassen sich so projizieren.{% endblocktrans %}


{% trans "Rednerliste" %}

{% blocktrans %}Jeder Tagesordnungspunkt enthält eine unabhängige Rednerliste, die von der Versammlungsleitung verwaltet wird. Teilnehmer können sich auch selbst auf die Liste setzen.{% endblocktrans %}

{% trans "Automatische Nummerierung" %}

{% blocktrans %}Auf Klick lässt sich die Tagesordnung automatisch nummerieren (mit arabischen oder römischen Ziffern); optional mit einstellbarem Präfix (z.B. "TOP 1").{% endblocktrans %}

{% trans "Zeitplanung" %}

{% blocktrans %}Schätzen Sie die Dauer für jeden Tagesordnungspunkt und erhalten Sie dadurch einen prognostizierten Endzeitpunkt der Tagung.{% endblocktrans %}

{% trans "Rednerliste auf zweiten Bildschirm" %}

{% blocktrans %}Behalten Sie die aktuelle Rednerliste über einen zweiten Bildschirm immer im Blick - während der Projektor zeitgleich für das Plenum den zugehörigen TOP anzeigt.{% endblocktrans %}

{% trans "Organisatorische Einträge" %}

{% blocktrans %}Neben formalen Tagesordnungspunkten kann eine Tagesordnung auch organisatorische Einträge (wie z.B. Pausen oder thematische Blöcke) enthalten.{% endblocktrans %}

{% trans "Erledigt markieren" %}

{% blocktrans %}Abgeschlossene Tagesordnungspunkte können als erledigt markiert und anschließend zur besseren Übersicht ausgeblendet werden.{% endblocktrans %}

{% trans "Tagesordnung als PDF" %}

{% blocktrans %}Die vollständige Tagesordnung lässt sich als PDF exportieren.{% endblocktrans %}





{% trans "Anträge" %}

  • ↑ {% trans "Nach oben" %}
{% trans "Anträge papierlos verwalten" %}

{% blocktrans %}Importieren Sie Anträge aus einer CSV-Datei. Legen Sie die automatische Nummerierungsart fest und bestimmen Sie die Zugriffsrechte für Anträge: Welche Gruppen dürfen Anträge sehen, erstellen, verwalten und unterstützen?{% endblocktrans %}

{% trans "HTML-Editor" %}

{% blocktrans %}Für einfache Formatierung von Antragstexten ist ein HTML-Editor ("CKEditor") integriert. Die Formatierungen werden zum Großteil auch in der PDF-Fassung unterstützt.{% endblocktrans %}

{% trans "Versionshistorie und Log" %}

{% blocktrans %}Änderungen eines Antrags können als neue Version gespeichert werden und lassen sich vergleichen. Jede Bearbeitungsaktion wird protokolliert und lässt sich zur besseren Rückverfolgung in OpenSlides einsehen.{% endblocktrans %}

{% trans "Unterstützen von Anträgen" %}

{% blocktrans %}Sie können einstellen, ob Unterstützer nötig sind, um einen Antrag zuzulassen. Teilnehmer können dann direkt in OpenSlides Ihre Zustimmung für einen Antrag äußern.{% endblocktrans %}

{% trans "Sachgebiete verwalten" %}

{% blocktrans %}Ordnen Sie Sachgebiete Ihren Anträgen zu und vergeben Sie zusätzlich ein passendes Präfix für die Antragsbezeichner, z.B. "SA 1".{% endblocktrans %}

{% trans "Antragsbuch als PDF" %}

{% blocktrans %}Alle Anträge lassen sich übersichtlich zu einem PDF zusammenfassen; inkl. Antragsteller, Abstimmungsergebnisse und Begründungen.{% endblocktrans %}

{% trans "Änderungsanträge erstellen" %} {% trans "Neu in 1.7" %}

{% blocktrans %}Zu einem Hauptantrag lässt sich auf Knopfdruck ein Änderungsantrag einreichen. Der Bezug zum Hauptantrag ist stets gegeben. Eine automatische Nummerierung hilft beim Filtern und Verwalten von Änderungsanträgen (z.B.: Hauptantrag L1 und zugehöriger Änderungsantrag L1 Ä1).{% endblocktrans %}

{% trans "Sichtbarkeit von Anträgen einschränken" %} {% trans "Neu in 1.7" %}

{% blocktrans %}Anträge lassen sich für bestimmte Teilnehmergruppen erst ab einem definierbaren Status öffentlich einsehen. D.h. normale Nutzer im Plenum sehen Anträge beispielsweise erst nach erfolgter Freigabe durch die Organisatoren. {% endblocktrans %}

{% trans "Schlagwörter zur Verwaltung" %} {% trans "Neu in 1.7" %}

{% blocktrans %}Versammlungsleiter können „Tags“ zur Verschlagwortung von Einträgen vergeben, um leichter inhaltlich zusammengehörige Elemente zu verwalten. Beispielsweise kann ein Antrag, ein Tagesordnungspunkt und eine Wahl mit dem gleichen Schlagwort versehen werden. {% endblocktrans %}

{% trans "Anträge abstimmen" %}
{% url 'support_packages' as supportpackages_url %}

{% blocktrans %}Nach Abstimmung eines Antrags im Plenum wird das ausgezählte Ergebnis in OpenSlides eingegeben (mit OpenSlides selbst wird nicht abgestimmt!). Dabei sind auch spezielle Werte für die Mehrheit der Stimmen, oder nicht erfasste Stimmen möglich. Anträge werden immer mit Ja-Nein-Enthaltung abgestimmt. Möglich ist auch ein Stimmzetteldruck für geheime Abstimmungen. Auf Anfrage sind elektronische Abstimmungen mit dem VoteCollector-Plugin möglich. {% endblocktrans %}

{% trans "Anträge in die Tagesordnung" %}

{% blocktrans %}Jeder Antrag lässt sich als Referenz zusätzlich in die Tagesordnung einsortieren.{% endblocktrans %}

{% trans "Arbeitsabläufe und Zustände definieren" %}

{% blocktrans %}Jeder Antrag durchläuft verschiedene Phasen in demokratischen Versammlungen. Nutzen Sie einen der zwei voreingestellten Arbeitsablä̈ufe mit vordefinierten Zuständen, die ein Antrag typischerweise einnehmen kann oder konfigurieren Sie Ihren individuellen Antragsablauf selbst.{% endblocktrans %}




{% trans "Wahlen" %}

  • ↑ {% trans "Nach oben" %}
{% trans "Wahlen verwalten" %}

{% blocktrans %}Jede Wahl besteht aus Name und Anzahl der zur Wahl stehenden Posten sowie - optional - aus einer Beschreibung des Amtes und einem Kommentar für den Stimmzettel.{% endblocktrans %}

{% trans "Personalisierte Stimmzettel" %}

{% blocktrans %}Geheime Wahlen benötgen Stimmzettel. Und diese können Sie unkompliziert direkt mit OpenSlides generieren - mit passenden Ankreuzfeldern für jeden Kandidaten.{% endblocktrans %}

{% trans "Unterstützte Wahlmethoden" %}

{% blocktrans %}OpenSlides unterstützt die Auswahl von 2 Wahlmethoden: "Eine Stimme pro Kandidat/in" oder "Ja, Nein, Enthaltung pro Kandidat/in". Eine automatische Zuordnung beider Methoden, abhängig von der Anzahl der Kandidaten und zu vergebenen Posten, ist die Voreinstellung in OpenSlides. Gültige, ungültige und abgegebene Stimmen können zusätzlich erfasst werden.{% endblocktrans %}

{% trans "Kandidatenliste aufstellen" %}

{% blocktrans %}Kandidaten können für eine Wahl direkt aus der Teilnehmerliste vorgeschlagen werden. Angemeldete Teilnehmer können per Mausklick sogar selbst kandidieren - sofern Sie die Berechtigung dazu haben.{% endblocktrans %}

{% trans "Wahlergebnisse eingeben und darstellen" %}

{% blocktrans %}Sind die Wahlergebnisse ausgezählt, werden sie in OpenSlides vom Versammlungsleiter eingegeben. Per Knopfdruck werden die Zahlen auch für alle anderen Teilnehmer sichtbar.{% endblocktrans %}

{% trans "Alle Wahlergebnisse in einem PDF" %}

{% blocktrans %}Exportieren Sie alle Wahlen ins PDF und geben Sie am Ende der Versammlung allen Delegierten eine Zusammenfassung aller gewählten Ämter. Oder nutzen Sie das PDF sogar als Anhang für Ihr Beschlussprotokoll.{% endblocktrans %}

{% trans "Mehrere Wahlgänge" %}

{% blocktrans %}Mit OpenSlides können Sie beliebig viele Wahlgänge durchführen - und diese übersichtlich nebeneinander darstellen. Auch Stichwahlen sind leicht damit möglich.{% endblocktrans %}




{% trans "Teilnehmer" %}

  • ↑ {% trans "Nach oben" %}
{% trans "Teilnehmer/innen verwalten" %}

{% blocktrans %}Nutzen Sie den CSV-Import zur Übernahme Ihrer vorhandenen Teilnehmerdaten. Dabei kann auch die Gliederungsebene, das Amt, die Benutzergruppe, ein Kommentar u.v.m. angegeben werden.{% endblocktrans %}

{% trans "Gruppen und Rechte festlegen" %}

{% blocktrans %}Was jeder Teilnehmer sehen darf und was nicht, definieren Sie mit einzelnen Rechten. Mehrere Rechte können Sie zu einer Gruppe zusammenfassen. Auch ein Gast (Anonymous-Nutzer) ist so aktivierbar.{% endblocktrans %}

{% trans "Zugangszettel für WLAN und OpenSlides" %}

{% blocktrans %}Verteilen Sie Ihren Teilnehmern beim Empfang einen generierte Infozettel mit Zugangsdaten für das WLAN am Veranstaltungsort und OpenSlides, inkl. praktischem QR-Code zum Abscannen.{% endblocktrans %}




{% trans "Allgemeines" %}

  • ↑ {% trans "Nach oben" %}
{% trans "Portable Version für Windows" %}

{% blocktrans %}OpenSlides ist für Windows als portable Version verfügbar. Seit 1.4 mit grafischem Frontend und komfortabler Einrichtung einer automatischen Backupfunktion der SQLite-Datenbank.{% endblocktrans %}

{% trans "Dokumentenablage" %}

{% blocktrans %}Nutzen Sie während Ihrer Versammlung eine gemeinsame Dokumentenablage für alle Teilnehmer. Regeln Sie, welche Gruppen Upload- und Download-Rechte bekommen sollen.{% endblocktrans %}

{% trans "PDF-Dokumente präsentieren" %}

{% blocktrans %}PDF-Dateien lassen sich in Vollbild an die Leinwand projizieren und zentral über ein Widget steuern. Damit können Sie nun auch in der letzten Reihe per Mobilgerät die Präsentation verfolgen.{% endblocktrans %}

{% trans "Gruppen-Chat" %}

{% blocktrans %}Nutzen Sie die Chatfunktion für einen reibungslosen Austausch unter den Organisatoren (z.B. Wahlleitung, Versammlungsleitung, Antragskommission und Regie).{% endblocktrans %}

{% trans "Volltextsuche" %}

{% blocktrans %}Finden Sie schnell den gesuchten Inhalt mit der integrierten (pythonbasierten) Volltextsuche - auch einschränkbar auf bestimmte Module. {% endblocktrans %}

{% trans "Responsive Design mit Bootstrap" %}

{% blocktrans %}OpenSlides erscheint im modernen, responsiven Design auf Basis von Bootstrap und ist optimiert für unterschiedliche Mobilgeräte.{% endblocktrans %}

{% trans "OpenSlides ist mehrsprachig" %}

{% blocktrans %}OpenSlides ist vollständig internationalisiert und ist derzeit in Deutsch, Englisch, Französisch, Tschechisch und Portugiesisch verfügbar. Weitere Sprachen können gerne über Transifex eingereicht werden.{% endblocktrans %}

{% trans "Plattformunabhängig mit Python" %}

{% blocktrans %}OpenSlides basiert auf dem Python-Webframework Django, ist plattformunabhängig auf Windows, MacOS und Linux lauffähig und integriert den leistungsfähigen Tornado Webserver. Als Datenbank wird voreingestellt SQLite verwendet (PostgreSQL und MySQL werden auch unterstützt).{% endblocktrans %}

{% trans "Projektortemplate anpassbar" %}

{% blocktrans %}Die Projektoransicht ist leicht an Ihre individuellen Gestaltungsrichtlinien anpassbar; über eine zentrale HTML- und CSS-Datei. Logo, Titel und Hintergrundfarbe ist sogar dirkt in der Oberfläche (unter Konfiguration) einstellbar.{% endblocktrans %}
